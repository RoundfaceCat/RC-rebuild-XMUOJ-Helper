# XMUOJ 自动做题机器人工厂 - 通用操作指南 (AI SOP)

> **致阅读本文件的 AI 助手（GPT, Claude, Grok, Cursor, Antigravity 等）**：
> 欢迎你。无论你是通过系统指令读到这里，还是用户直接让你阅读的，当你被要求“做 XMUOJ 的题”、“解决 JDxxx”或“跑某场竞赛”时，请**严格遵守**以下流水线规范。

## 一、 权限与确认法则 (极度重要)
1. **不要挤牙膏式地请求权限**！
2. **如果你是 Antigravity 等带有环境执行权的本地 Agent**：请在行动前，第一时间调用你的 `ask_permission` 或其他原生工具，**一次性向用户请求本项目目录的读写权限，以及执行终端命令 (`python`) 的权限**。
3. **如果你是纯对话模型 (ChatGPT / Claude Web)**：请在行动的第一句话告诉用户：“我需要您在本地执行以下脚本，请确认您已准备好。”
4. **绝不要在工作流程的中途反复停下来询问用户“接下来我要怎么做”**。按照下述流程全自动闭环。

## 二、 项目架构与业务逻辑
本项目是 XMUOJ 的本地答题流水线，核心结构如下：
- `problems.md`：包含所有题目的描述、输入输出样例。
- `answers/pending/`：你的**工作区**。所有生成的待提交代码（C/C++）必须保存在这里。
- `answers/correct/`：**荣誉区**。通过 OJ 判题 (Accepted) 的代码会自动被脚本移动到这里，你不需要手动移动。
- `xmuoj_bot.py`：核心的自动化脚本，用于抓取和提交。

## 三、 标准执行流 (Standard Workflow)
当用户给你指派了具体题目（例如 JD110）后，请按照以下步骤无限循环直至通过：

1. **审题**：
   从 `problems.md` 中找到对应题目。如果没有，可以考虑通过执行 `python xmuoj_bot.py --mode fetch` 抓取（如果支持的话），或直接向用户索要题意。
2. **编写代码 (仅限 C/C++)**：
   - 编写最高效、最低内存占用（**严格控制在 10MB 内存以内**）的 C/C++ 代码。
   - 避免频繁的动态内存分配（慎用频繁 `vector` 等，能开全局静态数组的全部开全局静态数组）。
   - 将代码保存至 `answers/pending/JDxxx.cpp`。
3. **执行提交**：
   在终端运行：`python xmuoj_bot.py --mode submit --contest 361 --language C++`（如果是 C 语言，则改为 `--language C`）。
4. **日志分析与错误自愈**：
   - 脚本运行结束后，仔细读取终端的 `[=] Final Status` 输出日志。
   - 如果是 `[+] [SUCCESS] Accepted!`：恭喜，任务完成！可以直接向用户汇报战果。
   - 如果是 `[-] [FAILED] ... Failed: Wrong Answer` 或 `Partial Accepted`，亦或是 `Runtime Error`：
     **不要向用户求助！** 直接读取它返回的判题日志，分析逻辑漏洞、边界溢出或者格式问题，重新修改 `answers/pending/JDxxx.cpp`。
   - 修改完毕后，**回到第 3 步**再次提交。
   - 最多允许失败重试 **3** 次。如果 3 次依旧无法通过，停止行动并向用户总结你的排错思路和遇到的瓶颈，请求人类支援。
