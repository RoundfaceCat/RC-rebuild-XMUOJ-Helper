# Coding Style Guidelines

1. Do not add comments anywhere EXCEPT for struct/linked list definitions and function definitions(DON'T write comments for the main fucntion). ALL comments MUST be written in Chinese (中文).
2. For variable naming, use snake_case (e.g., hash_map) instead of CamelCase or PascalCase (e.g., HashMap). Try to use English word abbreviations for naming, e.g., cur for current, pq for priority_queue, st for stack, num for number.
3. Insert some blank lines to logically separate different parts of the code and make it more readable.
